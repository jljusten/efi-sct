Welcome to the EFI Self-Certification Tests (EFI-SCT) Project on TianoCore. The EFI-SCT is a toolset for platform firmware developers to validate EFI implementations on IA-32 as well as Itanium® Architecture-based platforms for compliance with the EFI 1.10 Specification. The purpose is to ensure interoperability of different EFI implementations. The EFI SCT was released under the Eclipse license.

Working with the SCT

The toolset features a Test Harness for executing built-in EFI Compliance Tests, as well as for integrating user-defined tests that were developed using the EFI-SCT open source code. The main objective to EFI-SCT open source code is to provide standards for writing EFI 1.10 Specification compliant test cases, with shared output, reporting, and logging features. This allows test case writers to concentrate their efforts on test functionality.

The EFI-SCT requires the EFI Developer Kit (EDK) and the EFI-Shell to compile and build. Please refer the EDK Getting Started Guide (pdf) to install the EDK and the EFI Shell Getting Started Guide (pdf) to install the EFI Shell.

The following bullets provide information on how to download, build and use the EFI-SCT.

  • Download snaphots of development or official releases of the EFI-SCT. Simply click on the link and choose the latest development or official version to start the download process.
  • Download the EFI SCT Getting Started Guide (pdf) for guiding on building and using the EFI-SCT to develop.

Project Participation

The EFI-SCT project team is primarily comprised of Intel platform engineers, but the door is open to external engineers to join this blended team of professionals. Companies and individuals that may want to join the EFI-SCT development team include:

  • Independent Hardware Vendors (IHVs)
  • Independent BIOS Vendors (IBVs)
  • Original Equiment Manufacturers (OEMs)
  • Operating System Developers
  • Open BIOS Engineers
  • Educational Institutions
There is room for project membership in filing, screening, fixing and verifying bugs, as well as participating in the enhancement of the EFI-SCT . See the EFI-SCT Project Development Process for an explanation of how this project functions and how new users can participate.
